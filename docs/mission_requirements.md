# 3단계 - 의존성 리팩터링

- [x] 메뉴의 이름과 가격이 변경되면 주문 항목도 함께 변경
    - 메뉴 정보가 변경되더라도 주문 항목이 변경되지 않게 구현
- [x] 클래스 사이, 패키지 사이의 의존 관계는 단방향

### hint

- 함께 생성되고 함께 삭제되는 객체들을 함께 묶어라
- 불변식을 지켜야 하는 객체들을 함께 묶어라
- 가능하면 분리하라
- 연관 관계는 다양하게 구현할 수 있다.
    - 직접 참조 (객체 참조를 이용한 연관 관계)
    - 간접 참조 (리포지토리를 통한 탐색)
